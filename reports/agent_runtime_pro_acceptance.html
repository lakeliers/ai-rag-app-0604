本次升级后的主链路

Intent Classifier 意图分类器→ Planner 规划器→ Orchestrator 编排器→ DAG Runtime 有向无环图运行时→ Aggregator 聚合器→ Evaluator 评估器→ Final Answer 最终回答→ Answer Validator 答案校验器

代码级升级点

环节 升级前 升级后 对应文件
Intent Classifier 意图分类器 只输出 intent、confidence、reason。 补充 `suggested_action`、`entities`、`constraints`，给 Planner 提供结构化指引。 agent_runtime.py
Planner 规划器 只输出 action 和 reason。 补充 `confidence` 和 `params`，高层动作更像生产环境的结构化 plan。 agent_runtime.py
Orchestrator 编排器 直接返回 `list[AgentStep]`。 新增 `TaskNode`、`TaskGraph`，把高层动作翻译成任务图。 agent_runtime.py
DAG Runtime 有向无环图运行时 按列表顺序执行。 新增任务图校验、依赖判断、可执行批次、重试、关键/非关键节点失败策略。 agent_runtime.py
Aggregator 聚合器 只统计 upload/web/local 数量。 新增统一结构化、去重、来源配额、质量信号。 agent_runtime.py
Evaluator 评估器 只判断资料是否存在。 新增 `confidence`、`missing_aspects`、`next_action`，可给 Planner 下一步建议。 agent_runtime.py
Final Answer 最终回答 生成后直接返回。 新增 `answer_validator` trace，对空答案、引用提示、资料不足提示做规则校验。 agent_runtime.py
前端 Trace 展示 非 success 都显示为失败。 新增 warning/提示状态展示，避免答案校验提示被误标成失败。 app.py

验收结论

本次已经把课堂里的主流 Agent Runtime 结构落到代码中：Planner 不直接碰底层工具，Orchestrator 生成 TaskGraph，DAG Runtime 按依赖执行，Aggregator 做结果治理，Evaluator 给出充分性与下一步建议，Final Answer 生成后增加规则校验。当前实现仍是“轻量生产形态”，后续可以继续补真正异步并发、模型版 Evaluator、循环式 Observe-Act 和更完整的预算控制。
